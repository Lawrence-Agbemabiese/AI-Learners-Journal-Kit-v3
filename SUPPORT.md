# Support

This project supports two distribution modes: free/beta workshop use and paid digital product use.

## Free/Beta Workshop Support

Free/beta users should expect best-effort support through GitHub Issues or the workshop facilitator.

Good bug reports include:

- Operating system and version.
- Python version from `python --version` or `python3 --version`.
- The command that failed.
- The exact error message.
- Whether the user installed from a release ZIP, Git clone, or source download.

Do not include API keys, private notes, or sensitive journal entries in support requests.

## Paid Product Support

Paid customers should receive the support channel advertised on the storefront. A practical baseline is:

- Installation help for the current release.
- Bug fixes or workarounds for documented features.
- Clarification of documented workflows.
- Best-effort help with Python/path setup.

Paid support does not include custom automation, private tutoring, operating-system repair, or guarantees about third-party AI API output.

## Supported Platforms

The supported release target is:

- Python 3.9 or newer.
- macOS, Windows, or Linux.
- Local filesystem access to the user's home folder.

CI should pass on Ubuntu, macOS, and Windows before a paid release is published.

## Response Expectations

For paid storefronts, set a clear response expectation on the product page. A reasonable starting point is 2 business days for installation and product defects.

## Known Boundaries

The app is a local Markdown journal. It is not a cloud sync service, medical/legal/financial advisor, password manager, learning management system, or enterprise data platform.
